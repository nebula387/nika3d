# Nika 3D — голосовой 3D-компаньон

## Кто использует этот проект

Иван, личный pet-проект. С Python знаком, опыта с Unity/Godot/C#/GDScript
нет. Claude Code должен объяснять шаги установки и настройки подробно,
не предполагать знание игровых движков. Когда что-то нужно сделать руками
вне кода (установить программу, нажать кнопку в GUI) — давай пошаговую
инструкцию, не просто "настройте Godot".

**Язык общения с Иваном:** русский (комментарии в коде, консоль, объяснения).
**Язык самого пайплайна (ASR/LLM/TTS внутри brain/):** английский — жёстко,
см. "Решение: English-only pipeline" ниже. Это осознанный выбор Ивана,
не путать одно с другим.

## Что строим

Голосовой 3D-компаньон по имени Ника:
- Активация по имени ("Ника" / "Nika")
- Распознавание речи, LLM-диалог, синтез голоса
- НО: вместо Live2D-аватара (VTube Studio, старый проект) — полноценный
  3D-персонаж в Godot
- Персонаж не копирует пользователя (это не VTube/трекинг лица),
  а действует самостоятельно как игровой NPC
- По запросу пользователя — управление ПК голосом (открыть сайт/программу,
  напечатать текст, поискать в интернете)
- Долгосрочная цель: NPC-компаньон с расширяемым набором действий

## Железо пользователя

- Windows 11
- GPU: Intel Arc A580, 8GB VRAM (**НЕ Nvidia — критично!**)
- RAM: 16GB

## КРИТИЧЕСКОЕ ограничение архитектуры

GPU — Intel, не Nvidia. Это означает:
- НЕЛЬЗЯ использовать что-либо, требующее CUDA (Nvidia Audio2Face,
  TensorRT, любые CUDA-only ML библиотеки, torch с CUDA-сборкой)
- PyTorch стоит CPU-only. LM Studio использует Vulkan-бэкенд
  llama.cpp — это то, что реально даёт GPU-ускорение на Intel Arc
  (подтверждено: `lms.exe runtime survey` видит карту через Vulkan)
- Для XTTS (клонирование голоса) — тоже CPU (`gpu=False` в tts_xtts.py)
- Рендеринг 3D-сцены в Godot — должен быть на профиле Compatibility/Mobile

Если в любой момент предлагается решение — первым делом проверяй,
не тянет ли оно скрытую CUDA-зависимость.

## Архитектура: два процесса, один мост

### 1. Python — "мозг" (папка `brain/`)

- `main.py` — точка входа: запускает WebSocket-мост, загружает Whisper и
  TTS-голос, крутит основной цикл (слушать → LLM → говорить → действие)
- `asr.py` — Whisper: wake word + запись команды + follow-up.
  **Только английский** (см. решение ниже) — язык больше не определяется,
  везде `language="en"` жёстко
- `llm.py` — клиент к LLM (OpenAI-совместимый API). Роутинг запроса по трём
  веткам (см. "Три режима LLM" ниже), + память сессии (история диалога)
- `web.py` — живые данные из интернета: погода, курсы валют, новости/поиск
- `tts.py` + `tts_xtts.py` — синтез речи, провайдер выбирается в `.env`
- `actions.py` — управление ПК (открыть сайт/программу, напечатать текст) —
  **выполняется ЛОКАЛЬНО в Python, никогда не уходит в Godot** (см. ниже)
- `bridge.py` — WebSocket-сервер (Python=сервер, Godot=клиент), сейчас шлёт
  ТОЛЬКО `speech_state` (talking/idle) — это единственное, что Godot знает
  о происходящем

### 2. Godot — "тело" (папка `godot_project/`, живой проект — см. ниже)

Тонкий исполнитель: получает `speech_state` по WebSocket, переключает
Idle/Talking. Управление ПК (open_url/open_app/type_text) НЕ имеет
отношения к Godot — это чисто Python-side эффект, персонаж на экране
об этом не узнаёт.

> **Живой Godot-проект:** `C:\Godot\nika-3d\` (не внутри этой папки).
> Скрипты из `godot_project/scripts/` скопированы туда вручную.
> Persistент: `docs/setup-godot.md`.
> Источник 3D-персонажа: Quaternius `Casual.fbx` (не Mixamo — был недоступен),
> путь `C:\Godot\nika-3d\assets\character_final\Casual.fbx`.

### Три режима LLM (llm.py.ask())

Проверяются в этом порядке — важно не путать:

1. **Забыть контекст** — фразы вроде "forget everything" → сброс истории,
   без обращения к LLM
2. **Веб-запрос** (`_is_web_query`) — погода/курс валют/новости/явное
   "search online"/"check the internet". Погода и курс отвечают напрямую
   из API (без LLM — быстро, без риска выдумывания фактов). Новости и
   общий поиск идут через `ddgs` → короткий пересказ через LLM
3. **Команда действия** (`_is_action`) — "open ", "search ", "type " и т.п.
   → строгий JSON `{"reply":..., "action":{...}}`, действие выполняет
   `actions.py`
4. **Обычный разговор** (иначе) — plain text, с историей диалога

Все три пути после ответа пишут пару (вопрос, ответ) в общую историю
`llm._history`, так что разговор помнит контекст даже если предыдущая
реплика была командой/веб-запросом.

## Решение: English-only pipeline (2026-07-18/22, важно)

Иван явно попросил отказаться от русского/двуязычности во всём пайплайне
(ASR/LLM/TTS) в пользу максимальной точности и скорости распознавания
**его английской речи** (признался, что говорит с запинками, плохим
произношением). Причина: маленькие локальные модели (Whisper base,
qwen2.5-7b/gemma-3n) ненадёжно определяют язык сами — то путают язык
распознавания, то отвечают не на том языке, что тратит время и портит
качество. Английский жёстко зашит везде:

- `asr.py`: `language="en"` во всех вызовах `model.transcribe()`,
  `WAKE_VARIANTS` — только латиница (`nika`, `nica`, `nike`, `neeka`)
- `llm.py`: промпты прямо говорят "Reply ONLY in English", без
  определения языка
- `tts.py`: `TTS_PROVIDER=piper` (только английский голос) — выбран
  ради низкой задержки (реальный синтез ~0.2с против 2-10с у XTTS на CPU)

**Не откатывай это молча**, даже если разговор с Иваном идёт по-русски —
это про язык взаимодействия ASSISTANT с Иваном в Claude Code (русский),
а не про язык самого голосового пайплайна Ники (английский). Если Иван
попросит вернуть русский — это будет явный откат, спрашивай явно.

## Решение: LM Studio вместо только-Groq

`llm.py` поддерживает `LLM_PROVIDER=lmstudio` (локально, порт 1234) и
`LLM_PROVIDER=groq` (облако, запасной вариант). Сейчас основной — LM
Studio. `LLM_MODEL=auto` в `.env` — Ника сама спрашивает LM Studio через
расширенный API `/api/v0/models` (НЕ `/v1/models` — тот отдаёт вообще все
скачанные модели, а не только загруженную), какая модель сейчас реально
загружена, и использует её. Меняешь модель в LM Studio — .env трогать
не нужно.

## Решение: выбор TTS-провайдера

`TTS_PROVIDER` в `.env`: `piper` (сейчас, быстро, только EN) | `xtts`
(медленнее на CPU, но EN+RU и клонирование голоса — есть образец
`assets/voice_reference.wav`, склонированный из старого голоса
пользователя в ElevenLabs) | `elevenlabs` (облако, платно). XTTS требует
системный FFmpeg с DLL (НЕ просто pip-пакет) для декодирования
speaker_wav — если его нет, клонирование голоса упадёт с ошибкой
torchcodec/FFmpeg при загрузке, даже если синтез без клонирования работал.
