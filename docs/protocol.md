# Протокол моста Python ↔ Godot

Статус на 2026-07-22: реализован и используется только `speech_state`.
Это единственный источник правды по формату сообщений — если меняешь
формат в коде, обнови и этот файл в том же коммите.

**Важно:** управление ПК голосом (открыть сайт/программу, напечатать
текст — см. `actions.py`) реализовано в Этапе 1.6 и выполняется ПОЛНОСТЬЮ
на стороне Python (через `webbrowser`/`subprocess`/`pyautogui`). Эти
команды НЕ отправляются в Godot и никак не описаны в этом протоколе —
персонаж на экране не узнаёт, что открылся браузер. Раздел "action" ниже
описывает изначально запланированный, но так и не реализованный механизм
(проигрывание анимаций типа wave/nod по команде) — не путать с
управлением ПК.

## Транспорт

WebSocket. Python — сервер, Godot — клиент.
Python запускается первым и занимает порт; Godot переподключается каждые 2 с.

Адрес: `ws://localhost:8765` (BRIDGE_HOST/BRIDGE_PORT в .env)

## Направление: Python → Godot

Все сообщения — JSON-объект с полем `type`.

### speech_state (Этап 1)

Сигнализирует, говорит ли сейчас Ника.

```json
{"type": "speech_state", "state": "talking"}
```
```json
{"type": "speech_state", "state": "idle"}
```

### visemes (не реализовано)

Запланировано, но НЕ сделано (см. `CLAUDE.md` → "Поэтапный план",
пункт "не начато"). Временная шкала визём для покадровой анимации рта,
синхронизированная с проигрыванием WAV-файла на стороне Python.

```json
{
  "type": "visemes",
  "data": [
    {"time": 0.0,  "shape": "X"},
    {"time": 0.12, "shape": "B"},
    {"time": 0.24, "shape": "C"}
  ]
}
```

`shape` — буквенный код виземы Rhubarb (A-H, X). Соответствие
код → форма рта/blend shape фиксируется в `character_controller.gd`.

### action (не реализовано — не путать с управлением ПК, см. выше)

```json
{"type": "action", "name": "wave"}
```
